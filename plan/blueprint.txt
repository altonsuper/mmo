2. Runtime World State Layer

Create a separate live-world runtime state manager distinct from static asset definitions.

The runtime world manager must:

track active spawned entities,
maintain unique instance IDs,
support coordinate positioning,
and allow runtime mutation independently from base templates.

Example runtime concepts:

entity spawn coordinates,
current HP,
temporary buffs,
ownership references,
despawn timers,
movement states.

Implement lightweight Area-of-Interest simulation patterns:

only render nearby active entities,
structure the grid system for future chunk partitioning,
include comments explaining scalability upgrades.

Do NOT tightly couple rendering logic to the data registry.
